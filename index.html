Trabalho - Git e GitHub

1. Conceitos de Git e GitHub

Git

É um sistema de controle de versão distribuído, criado por Linus Torvalds em 2005. Ele permite salvar o histórico de alterações de um projeto, trabalhar em paralelo em diferentes versões (branches) e colaborar em equipe.

Comandos principais: git init, git add, git commit, git branch, git merge, git log.

GitHub

É uma plataforma online que hospeda repositórios Git. Permite colaboração, abertura de issues, pull requests e deploys (como o GitHub Pages).

2. Fluxo de trabalho no Git/GitHub

  • main: versão estável do projeto.
  • develop: recebe funcionalidades antes de irem para produção.
  • feat/…: branch para cada nova funcionalidade.
  • Pull Request (PR): revisão e integração do código.
  • Merge: integração com develop e depois main.

Exemplo: Criar branch feat/pagina-home → fazer commits semânticos → abrir PR para develop → revisar → integrar no main.

3. Como resolver conflitos

Conflitos acontecem quando duas pessoas alteram a mesma parte de um arquivo.

  1. O Git marca no arquivo com <<<<<<< e >>>>>>>.
  2. O desenvolvedor escolhe a versão correta ou une as duas.
  3. Executa: 
    git add arquivo
git commit -m "fix: resolvido conflito"

4. Boas práticas

Commit Semântico

  • feat: adiciona página de contato
  • fix: corrige bug de responsividade
  • docs: adiciona explicação no README
  • style: ajusta espaçamento do CSS
  • refactor: melhora função de validação

Branches com tipo

  • feat/: novas funcionalidades
  • fix/: correções de bugs
  • hotfix/: correções urgentes em produção

Deploy no GitHub Pages

  1. Subir o arquivo index.html para o repositório.
  2. Ir em Settings → Pages.
  3. Escolher a branch main e pasta /root.
